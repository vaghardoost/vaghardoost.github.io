بعضا شاید اینطور تصور بشه که برنامه نویس های حرفه ای تر backend کارها هستن.البته که backend بسیار مهم و غیرقابل انکار هست اما دنیای فرانت برای من جای عجیبی بود

بک اند کجا فرانت کجا؟

بیشتر توی backend چالش هایی که داشتم بحث performance بود. جایی که سیستم نمیکشید یا باید روش رو عوض میکردیم یا باید سخت افزار تهیه کنیم. چون به قول معروف مغز متفکر و عامل اجرا backend هست یا فرانت باید از قانون backend تبعیت کنه.

شما برای backend با جاوا حال نمیکنید با سی شارپ کد بزنید، به گروه خونی شما نمیخورد، php هست ، املای گنگ و دشوار براتون داره ، سراغ پایتون برید یا روبی یا کلی زبان دیگه هست و برای هرکدوم اینا کلی فریم ورک هست.ولی اگر وب سایت میخواید ، چاره ای جز html و css و javascript ندارید. هرچند برای اینها فریم ورک هایی هست ولی درنهایت نمیتونید بدون دونستن برای مثال html تمام یک سایت را بسازید. خلاصه وار اینکه بدون جاوا یا سی شارپ میشه سایت ساخت بدون html نه.

تسک backend و frontend

از نظر تعداد تسک ها یا وظایف برای اون کاری که من انجام میدادم تسک فرانت 10 برابر backend بود.

چون معمولا backend برای یکبار نوشته میشه و دست آخر هرچقدرم کمالگرا باشید ، به اون فرم نهایی و دلخواه میرسید. اما فرانت بعد یه مدت فرانت حوصله کاربر رو سر میبره و باید یه تغییراتی رو توش اعمال کرد.

کد فرانت عموما روی سیستم کاربر نهایی اجرا میشه و منطق میگه برای کمترکردن فشار روی سیستم بیشتر کار رو روی فرانت انجام بده. برای مثال فرانت یک قالب حاضر و آماده از json رو باید از backend تحویل بگیره تحلیل کنه و نمایش بده. یا برعکس داده ای از کاربر بگیره و اعتبار سنجی کنه و داده مطمئن برای backend بفرسته. توی همچین سناریو هایی backend کلا یه crud بیشتر نیست. ولی فرانت علاوه بر نمایش داده به شکل چشم نواز ، زحمت همه کارهای جزیی رو باید بکشه.

تسک backend ممکنه برای توسعه پذیری احتیاج باشه شما کد به قول معروف تمیز بزنید. خب این چیزیه که فرانت هم میخاد.

نود جی اس این امکان را آورد که برنامه نویسان فرانت هم بدون نیاز به یادگیری زبان برنامه نویسی دیگری ، backend بنویسند.

نتیجه گیری

البته برنامه نویسی فرانت تاثیراتی هم بر روی برنامه نویسی backend گذاشت. از جمله اینکه قالب json که بومی زبان جاوا اسکریپت هست زبان انتقال داده های بسیاری از وب سرور ها شد و حتی پایگاه های داده ای مثل مانگو دی بی بوجود اومد که نوع جدیدی از پایگاه های داده رو معرفی کرد.

در مجموع نه backend به تنهایی به جایی میرسد و نه frontend. اما هرگز برنامه نویسان فرانت و طراح های وب را دستکم نگیرید. اون دسته برنامه نویسی که کدش باید هم باید حواسش به گرافیک و ظاهر باشد هم به محتوا و هم تحلیل کند برنامه نویسی خبره هست.
